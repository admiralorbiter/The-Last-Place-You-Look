# Vision

## Product name
**The Last Place You Look**

## One-sentence product statement
A local-first desktop app that helps people understand, organize, compare, and safely manage files spread across mounted local drives, starting with messy archives on external storage.

## Product category
Early on, the product is best described as a **personal storage operating system / media archive tool**. Over time it may grow into a broader personal storage operating system, but MVP is centered on archive clarity, duplicate resolution, and low-friction file management.

## Core promise
The product gives users **clarity and control** over their digital mess.

It should help users:
- see what they have
- understand how files relate
- compare copies and versions safely
- organize with low friction
- come back later and still understand what happened

## What it is not
This is not primarily:
- a cloud backup service
- a generic file explorer replacement
- a team collaboration tool
- a media streaming server
- a hard-delete cleanup utility

## Differentiators
The product is differentiated by combining:
- unified visibility across mounted local volumes
- duplicate intelligence with explanations
- protection awareness based on user-visible rules
- relationship views between assets, copies, and storage locations
- cautious, reversible file actions
- a calm interface with progressive disclosure

## MVP focus
MVP focuses on:
- mounted local storage sources, especially removable external drives
- staged scanning and unified cataloging
- duplicate comparison and guided resolution
- virtual organization plus guided real move/merge actions
- per-drive quarantine instead of hard delete
- relationship and storage maps at the item/group level
- rule-based protection awareness

## Emotional design target
The user should feel:
- less overwhelmed
- less afraid of making mistakes
- more confident that their archive makes sense
- able to return later without re-learning the mess

## Design principles
- Local-first by default
- Reversible before destructive
- Explain uncertainty clearly
- Do not pretend “protected” means “backed up” unless a visible rule says so
- Preserve user trust over automation theatrics
- Reveal complexity progressively
- Respect the filesystem rather than hiding it

## Non-goals for MVP
- cloud sync
- backup provider integrations
- network/NAS-first workflows
- semantic AI search
- automatic deletion
- a giant all-library node graph

## Success criteria for MVP
MVP is successful when a target user can:
1. attach one or more drives and create a unified catalog
2. identify duplicate groups and compare likely best copies
3. understand where copies live and which files are at risk
4. move/merge files with confidence, without hard deletion
5. return later and still understand the archive state
